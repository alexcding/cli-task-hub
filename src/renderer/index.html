Polling

TaskHub has no live connection to GitHub or Jira, so it checks on a timer instead. These are two independent fetch loops — GitHub PRs change far more often than Jira tickets, and each CLI hits a different rate-limited API — so they have their own intervals.

How often gh re-checks each project for PR/CI changes, new review requests, and merges. The GitHub API is rate-limited (5,000 requests/hour for an authenticated user), so polling too often across many projects can exhaust your quota. Lower = fresher data but more API calls. Minimum 15s.

How often acli re-runs your Jira queries. Tickets move slower than PRs, so this defaults higher. Minimum 30s.
